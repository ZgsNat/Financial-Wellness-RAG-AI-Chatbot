# FinWell Frontend

Next.js 16 frontend for the **Financial Wellness & Mental Health** application.

---

## Prerequisites

| Tool | Version |
|------|---------|
| Node.js | >= 20 |
| npm | >= 10 |
| Backend running | Docker Compose up |

---

## Quick start

```bash
# 1. Install dependencies (already done if you ran setup)
npm install

# 2. Copy env file (already created, verify API URL)
cat .env.local
# NEXT_PUBLIC_API_URL=http://localhost:8000

# 3. Start development server
npm run dev
```

Open [http://localhost:3000](http://localhost:3000)

> The backend must be running first. Start it with:
> ```bash
> cd ../backend
> docker compose up -d
> ```

---

## Running with the full stack

```bash
# Terminal 1 — start backend (Kong gateway on :8000)
cd backend
docker compose up -d

# Terminal 2 — start frontend
cd frontend
npm run dev
```

| URL | Service |
|-----|---------|
| http://localhost:3000 | Frontend (Next.js) |
| http://localhost:8000 | Backend API gateway (Kong) |
| http://localhost:8001 | Kong Admin API |
| http://localhost:15672 | RabbitMQ management |
| http://localhost:16686 | Jaeger tracing |

## Pages

### `/login` & `/register`
- Public, no auth required
- `login` → POST `/auth/login` → stores JWT in Zustand + cookie

### `/dashboard`
- Summary: total expenses, income, avg mood for current month
- Spending by category pie chart
- Recent transactions list
- Latest AI insight card

### `/transactions`
- Paginated table (20/page) with date filter
- Add / Edit / Delete via dialog forms
- Supports `expense` / `income` types, 8 categories

### `/journal`
- Mood logger (1–5 emoji scale) with optional note
- Journal entries: create, edit, delete with live word count
- Recent moods strip (last 7)

### `/chat`
- Real-time AI chat via SSE (`/insights/chat`)
- Messages stream token-by-token
- Sources displayed after response
- Rate limit: 10 messages/min (enforced by backend)

### `/notifications`
- Lists alerts generated by backend (spending spikes, wellness tips…)
- Mark as read, filter unread-only
- Auto-polls every 30 seconds; badge count in sidebar

## Auth flow

1. User registers/logs in → receives JWT access token
2. Token stored in Zustand (persisted in `localStorage` as `fw-auth`)
3. Token also written to a `fw-token` cookie for the proxy route guard
4. Axios interceptor attaches `Authorization: Bearer <token>` to every request
5. On 401 → auto-logout + redirect to `/login`
6. SSE chat uses `fetch` directly (not Axios) because streaming requires manual `ReadableStream` handling

## Troubleshooting

**"Failed to load" / 401 errors**
- Ensure backend is running: `docker compose ps` in `/backend`
- Verify Kong is healthy: `curl http://localhost:8000/auth/login`
- Clear browser storage and log in again

**Chat not streaming**
- Check browser DevTools → Network → `/insights/chat` — should be `text/event-stream`
- Ensure the insight service and embedding service are both healthy

**CORS errors**
- Kong does not add CORS headers by default in this config
- For local dev both apps run on different ports; add CORS plugin to Kong or use a Next.js API route as proxy if needed
